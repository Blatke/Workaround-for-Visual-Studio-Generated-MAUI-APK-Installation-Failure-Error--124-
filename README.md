## 0. Environment
- Visual Studio 2026 18.7.0
- Xiaomi HyperOS 3.0.302.0 based on Android 16
- Windows 10 Home Edition 22H2 19045.7291

## 1. Cause of the Issue

When you attempt to install a `.apk` file generated by Visual Studio (either directly built or via the "Distribute" wizard) onto a Xiaomi HyperOS 3.0 (based on Android 11+) phone, the following error occurs:

`-124: Failed parse during installPackageLI: Targeting R+ (version 30 and above) requires the resources.arsc of installed APKs to be stored uncompressed and aligned on a 4-byte boundary`

**Root Cause**:

Starting from Android 11 (API 30), the system strictly requires that the `resources.arsc` file inside the APK package must meet two conditions:

* **Uncompressed** (Stored mode, not Deflate compressed)
* **4-byte aligned** (The starting offset address of the file within the APK must be a multiple of 4)

During its default publishing workflow (even in Release configuration), Visual Studio bundles the APK using the `aapt2` tool, which compresses `resources.arsc` by default (Deflate). This results in a generated APK that fails to comply with Android 11+ installation requirements, causing it to be rejected by the HyperOS system.

## 2. Solution Strategy

Since adding configurations like `<AndroidNoCompress>resources.arsc</AndroidNoCompress>` to the project file (`.csproj`) did not take effect in your environment (potentially overridden by the MAUI build system or due to toolchain issues), we will adopt a **manual post-processing** approach:

* Extract the APK generated by VS
* Delete the original signature folder (META-INF)
* **Repack using 7-Zip**: Add `resources.arsc` using the **Uncompressed (Store)** method, while compressing other files normally
* Perform a **4-byte alignment** (zipalign) on the newly generated APK
* **Resign** the APK using your original keystore
* Finally, install it via ADB

This approach bypasses the VS build bugs and guarantees that the final APK fully complies with system requirements.

## 3. Operational Steps (Creating and Executing a BAT Batch File)

### Prerequisites

* [7-Zip](https://www.7-zip.org) installed (Default path: `C:\Program Files\7-Zip\7z.exe`)
* [Android SDK](https://developer.android.com/studio) Build Tools installed (containing `zipalign` and `apksigner`), your path is:
`C:\Users\YourUserName\AppData\Local\Android\Sdk\build-tools\37.0.0`
* An available keystore file (e.g., `KeyName.keystore`), with the known alias `keyname`
* The original APK file generated by VS named `VS_Generated.apk`, stored in the `D:\APKs` folder

### 3.1 Create the Batch File

Create a new text file inside the `D:\APKs` folder, name it `FixAPK.bat`, right-click and select "Edit", then copy and paste the following content:

```batch
@echo off
title APK Repair Tool - Resolving resources.arsc Compression Issue

:: Set working directory (Modify according to your actual setup)
set WORK_DIR=D:\APKs
set ORIGINAL_APK=%WORK_DIR%\VS_Generated.apk
set OUTPUT_FINAL=%WORK_DIR%\final.apk

:: 7-Zip Path
set SEVENZIP="C:\Program Files\7-Zip\7z.exe"

:: Android SDK Build Tools Path (Modify according to your actual path)
set BUILD_TOOLS=C:\Users\YourUserName\AppData\Local\Android\Sdk\build-tools\37.0.0

:: Keystore Information
set KEYSTORE_PATH=C:\Users\YourUserName\AppData\Local\Xamarin\Mono for Android\Keystore\KeyName\KeyName.keystore
set KEY_ALIAS=keyname

cd /d %WORK_DIR%

:: Check if the original APK exists
if not exist "%ORIGINAL_APK%" (
    echo Error: Cannot find the original APK file "%ORIGINAL_APK%"
    pause
    exit /b 1
)

:: Check for 7-Zip
if not exist %SEVENZIP% (
    echo Error: 7-Zip not found. Please install it and verify the path.
    pause
    exit /b 1
)

:: Clean up temporary directory
rmdir /s /q temp_repack 2>nul

:: 1. Extract the original APK
echo [1/6] Extracting %ORIGINAL_APK% ...
%SEVENZIP% x "%ORIGINAL_APK%" -otemp_repack -y >nul

:: 2. Remove old signature
echo [2/6] Removing old signature files...
rmdir /s /q temp_repack\META-INF 2>nul

:: 3. Repack (resources.arsc uncompressed, other files maximum compression)
echo [3/6] Repacking (Forcing resources.arsc to Store mode)...
cd temp_repack
:: Add the uncompressed resources.arsc first
%SEVENZIP% a -tzip -mx=0 ..\repacked_unaligned.apk resources.arsc >nul
:: Then add all other files (excluding resources.arsc) using maximum compression
%SEVENZIP% a -tzip -mx=9 ..\repacked_unaligned.apk * -x!resources.arsc >nul
cd ..

:: 4. Align APK
echo [4/6] Executing 4-byte alignment...
"%BUILD_TOOLS%\zipalign" -v -p 4 repacked_unaligned.apk aligned.apk >nul

:: 5. Sign APK
echo [5/6] Signing with keystore...
"%BUILD_TOOLS%\apksigner" sign --ks "%KEYSTORE_PATH%" --ks-key-alias %KEY_ALIAS% --out "%OUTPUT_FINAL%" aligned.apk
if errorlevel 1 (
    echo Signing failed! Please check if your keystore password is correct.
    pause
    exit /b 1
)

:: 6. Clean up temporary files
echo [6/6] Cleaning up temporary files...
del repacked_unaligned.apk aligned.apk 2>nul
rmdir /s /q temp_repack 2>nul

echo ========================================
echo Fix completed! Final APK generated: %OUTPUT_FINAL%
echo ========================================
pause

```

> [!NOTE]
>
> In the code given above, please modify the variables in the script based on your actual SDK path, keystore path, and alias.
> ```
> set BUILD_TOOLS=C:\Users\YourUserName\AppData\Local\Android\Sdk\build-tools\37.0.0
> set KEYSTORE_PATH=C:\Users\YourUserName\AppData\Local\Xamarin\Mono for Android\Keystore\KeyName\KeyName.keystore
> set KEY_ALIAS=keyname
> ```

### 3.2 Execute the Batch File

* Copy the VS-generated `VS_Generated.apk` file into the `D:\APKs` folder (or generate it directly in that directory).
* Double-click to run `FixAPK.bat` (or right-click and choose "Run as administrator").
* The script will display prompts for each step during execution, taking about 10 to 20 seconds to complete.
* If a password is required during signing, the command line will prompt: `Keystore password for signer #1:`. Type your password manually (it will not be displayed on the screen) and press Enter.

**Execution Result**:

A file named `final.apk` will be generated in the `D:\APKs` folder. The `resources.arsc` inside this file is now **uncompressed** (you can verify this using 7-Zip, where the Method column will show `Store`), fully aligned, and signed.

## 4. Installing the Generated APK to the Phone via CMD

1. Open Command Prompt (CMD) or PowerShell.
2. Navigate to the directory containing the final APK:
```cmd
cd /d D:\APKs

```


3. Ensure your phone is connected to the computer via USB and "USB debugging" is enabled under "Developer options".
4. Execute the installation command:
```cmd
adb install -r final.apk

```


(The `-r` parameter means overwrite if the app is already installed)
5. Wait for the installation to finish. The phone screen will display "App installed successfully" or the CMD will output `Success`.

**Verification**:

Open the application on your phone and verify that it functions correctly. If you need to update the app in the future, simply repeat the batch process above (using the newly generated VS APK as `VS_Generated.apk`) and install the `final.apk` again.

## 5. Appendix: Long-Term Recommendation

While manual fixing works, to save time it is highly recommended to try fixing the project file permanently. Add the following to the first `PropertyGroup` section of your `.csproj` file:

```xml
<AndroidAapt2AdditionalArguments>--no-compress resources.arsc</AndroidAapt2AdditionalArguments>
<AndroidNoCompress>resources.arsc</AndroidNoCompress>

```

After doing this, delete the `bin` and `obj` directories, then rebuild the APK. If the generated APK is still compressed, please report the issue to the .NET MAUI community and continue using this batch script as a workaround.

**Summary**: By following these steps, you can transform any Visual Studio-generated APK into an installable package that complies with Android 11+ requirements without relying on correct VS configurations, successfully running your application on HyperOS 3.0.
